GIT HELP

  1. Configuration and Setup

    git config

    Function: Sets user information and preferences.

    Common Variations:

    git config --global user.name "Your Name"

    git config --global user.email "you@example.com"

    git config --list (view config)

    When to use: First-time setup or when changing Git identity/preferences.

  2. Creating and Cloning Repositories

    git init

    Function: Initializes a new Git repository in the current folder.

    When to use: Starting a new project under version control.

    git clone [SSH]

    Function: Creates a local copy of a remote repository

    Variations

    git clone [url] [directory] (clone into a specific folder)

    git clone --branch [branch] [url] (clone only a specific branch)

    When to use: Begin working on an existing project from GitHub, GitLab, etc.

  3. Checking Status and History

    git status

    Function: Shows Changes in working directory, staged changes, and untracked files.

    When to use: Before staging, committing, or pushing.

    git log

    Function: Displays commit history.

    Variations:

    git log --oneline (compact view)

    git log --graph --oneline --decorate (branch visualization)

    When to use: Reviewing history or identifying commits.

    git diff

    Function: Shows differences between working files, staging area, or commits.

    Variations:

    git diff (working directory vs. staging)

    git diff --staged (staging vs. last commit)

    git diff [commit1] [commit2]

    When to use: Checking what has changed befor committing.

  4. Staging and Committing

    git add

    Function: Stages files for commit.

    Variations:

    git add [file] (single file)

    git add . (all changes in current directory)

    git add -p (interactive staging by hunk)

    When to use: Before committing changes.

    git commit

    Function: Saves staged changes to history.

    Variations:

    git commit -m "message"

    git commit -a -m "message" (stages and commits tracked files)

    git commit --amend (edit last commit)

    When to use: To record changes, amend mistakes, or update messages.

  5. Branching and Merging

    git branch

    Function: Lists, creates, or deletes branches.

    Variations:

    git branch (list branches)

    git branch [name] (create branch)

    git branch -d [name] (delete branch)

    When to use: Managing different lines of development.

    git checkout

    Function: Switches branches or restores files.

    Variations:

    git checkout [branch] (switch to branch)

    git checkout -b [branch] (create and switch)

    git checkout [file] (restore file)

    When to use: Moving between branches or undoing file changes.

    git switch

    Function: Modern alternative to checkout for branch switching.

    Variations:

    git switch [branch]

    git switch -c [branch] (create and switch)

    When to use: Prefef over checkout when switching branches.

    git merge

    Function: Combines changes from one branch into another.

    When to use: Integrating completed feature branches.

    git rebase

    Function: Moves commits to a new base.

    When to use: Keeping commit history linear. Use with caution on shared branches.

  6. Remote Repositories

    git remote

    Function: Manages remote repository connections.

    Variations:

    git remote -v (list remotes)

    git remote add [name] [url]

    When to use: Linking local repo to remotes.

    git fetch

    Function: Downloads remote changes without merging.

    When to use: To preview remote updates before merging.

    git pull

    Function: Fetches and merges remote changes.

    Variations:

    git pull --rebase (fetch and rebase instead of merge)

    When to use: Keeping local branch updated with remote.

    git push

    Function: Uploads local commits to remote.

    Variations:

    git push origin [branch]

    git push -u origin [branch] (set upstream branch)

    git push --force (overwrite remote history, dangerous)

    When to use: Sharing commits with others.

  7. Undoing and Fixing

    git reset

    Function: Moves HEAD to a different commit and optionally updates the index (staging area) and working directory.

    Common Variations:

    git reset --soft [commit] (move HEAD only; keep changes staged)

    git reset --mixed [commit] (default — move HEAD and unstage changes, changes remain in working dir)

    git reset --hard [commit] (move HEAD and discard all working-directory and staged changes)

    When to use: To undo or rewrite local commits before they are pushed. Be careful with --hard because it permanently discards changes.

    git revert

    Function: Creates a new commit that reverses the changes introduced by an earlier commit.

    Common Variations:

    git revert [commit] (revert a single commit)

    git revert --no-commit [commit] (apply the revert but leave changes staged so you can bundle multiple reverts into one commit)

    When to use: To safely undo changes in shared/public history — it preserves history by adding a new commit that undoes the specified commit.

    git stash

    Function: Temporarily saves (stashes) uncommitted changes so you can work on a clean tree, then reapply them later.

    Common Variations:

    git stash (stash tracked modifications and staged changes)

    git stash save "message" (stash with a description; older syntax)

    git stash list (show stashed entries)

    git stash pop (apply the most recent stash and remove it from the stash list)

    git stash apply (apply a particular stash without removing it)

    git stash drop (delete a specific stash)

    When to use: When you need to switch branches or pull changes but aren’t ready to commit your current work. Use pop to reapply and remove the stash, or apply if you want to keep it for later reuse.

  8. Advanced/Helpful Commands

    git tag

    Function: Marks specific commits as releases.

    Variations:

    git tag [name]

    git tag -a [name] -m "message"

    git push origin [tag]

    When to use: Versioning and release management.

    git cherry-pick [commit]

    Function: Applies a specific commit onto the current branch.

    When to use: Bringing over single changes without merging branches.

    git reflog

    Function: Shows history of all HEAD movements.

    When to use: Recovering lost commits after resets or mistakes.

    git clean

    Function: Removes untracked files.

    Variations:

    git clean -n (preview)

    git clean -f (force delete)

    When to use: Cleaning up a working directory.

  9. Comparison: When to Use What

    merge VS. rebase: Use merge for preserving full branch history and use rebase for a cleaner, linear history.

    reset VS. rebase: Use reset for local undo (before pushing); use revert for undoing changes in shared history.

    checkout VS. switch: Prefer switch for branch changes; use checkout when restoring files.

    pull VS. fetch: Use fetch to preview changes
